# Лабораторная работа 4

## Что сделано по текущему сценарию

- Сохранен старый главный экран сайта (`index.html`) в стиле lab1.
- В хедер добавлен отдельный пункт `ADMIN`.
- Добавлена отдельная страница `admin.php`:
  - регистрация,
  - вход,
  - выход,
  - админка списка лабораторных.
- Список лабораторных хранится в `storage/labs.json`.
- Добавление/удаление лабораторных выполняется через AJAX (`fetch`) и `api.php`.
- При добавлении лабораторной изображение загружается файлом и сохраняется в папку `img/`.
- Страница `labs.html` теперь берет карточки лабораторных из `storage/labs.json` (включая ссылку и картинку).

## Ключевые файлы

- `index.html` — старый основной сайт + пункт `ADMIN` в хедере.
- `admin.php` — страница входа/регистрации и управления лабораторными.
- `api.php` — API и сессии (`status`, `register`, `login`, `logout`, `labs_list`, `labs_add`, `labs_delete`, `labs_public`).
- `lab4.js` — клиентская логика авторизации и админки.
- `lab4.css` — стили админ-страницы в эстетике основного сайта.
- `storage/users.json` — пользователи.
- `storage/labs.json` — лабораторные.
- `labs.html` — отображение лабораторных карточек из `labs.json`.

## Как проверить

1. Открыть `index.html`, убедиться, что визуально это старый сайт.
2. Перейти по пункту `ADMIN`.
3. Зарегистрировать `admin` (первый пользователь `admin` получает роль администратора).
4. Войти под `admin`.
5. Добавить лабораторную и выбрать файл картинки.
6. Убедиться, что:
   - запись появилась в таблице админки,
   - файл картинки появился в `img/`,
   - в `storage/labs.json` сохранился путь `img/...`.
7. Открыть `labs.html` и проверить, что новая карточка лабораторной отображается с картинкой.
